# Contributing to Docs

## Rules
- Keep docs aligned with actual code and generators.
- Update affected docs in same change when behavior changes.
- Avoid speculative mechanics without implementation note.

## Required Updates by Change Type
- Gameplay logic changes: update `GAMEPLAY_SYSTEMS.md` and related UML.
- Build/tool changes: update `CONTENT_PIPELINE.md`.
- Story/world generation changes: update `WORLD_STORY_BIBLE.md` and `GDD.md`.

## Review Checklist
- Terminology consistent across docs.
- Counts and IDs match generated artifacts.
- Command examples run on current repo layout.
